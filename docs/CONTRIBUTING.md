# Contributing Guide

본 문서는 우리 팀의 Git 커밋 컨벤션 규칙을 설명합니다.  
PR 및 코드 리뷰 전 반드시 확인해주세요.

---

## 1. 커밋 유형 지정
커밋 유형은 영어 소문자로 작성합니다.

| 커밋 유형 | 의미 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `style` | 코드 formatting, 세미콜론 누락, 코드 자체의 변경이 없는 경우 |
| `refactor` | 코드 리팩토링 |
| `chore` | 패키지 매니저 수정, 그 외 기타 수정 (예: .gitignore) |
| `test` | 테스트 코드, 리팩토링 테스트 코드 추가 |
| `design` | CSS 등 사용자 UI 디자인 변경 |
| `comment` | 필요한 주석 추가 및 변경 |
| `rename` | 파일 또는 폴더 명만 수정/이동 |
| `remove` | 파일 삭제 |
| `!BREAKING CHANGE` | 커다란 API 변경 |
| `!HOTFIX` | 긴급 치명적 버그 수정 |
| `WIP` | 작업 중(Work In Progress) → 완료 시 `git rebase -i` 또는 `git commit --amend`로 정리 |

---

## 2. 제목과 본문 구분
- 제목과 본문은 **빈 줄로 분리**
- 제목: `타입: 한글 설명` 형태 사용
- 제목은 **50자 이내 권장**
- 본문 각 행은 **72자 이내 권장**

---

## 3. 본문 작성
- 변경한 **무엇(What)**과 **이유(Why)** 설명  
- **어떻게(How)** 보다는 의도와 맥락 중심  
- 파일/폴더 단위 변경사항은 bullet(`-`)로 정리

예시:
```text
feat: 카테고리 기반 동적 UI 및 콘텐츠 구조 개선

헬스리포트의 시각적 이해도를 높이고 콘텐츠를 구조화하기 위해
카테고리 키를 추가하고, 각 건강 지표와 생활습관에 맞는
아이콘을 동적으로 표시하도록 UI 개선함.

- config/: 중요_지표_목록, 생활습관_목록에 카테고리 필드 추가
- models/: Pydantic 모델 구조 업데이트
- template/: 카테고리 값 기반 img src 동적 생성
- style.css: 변경된 UI 구조에 맞춘 레이아웃 및 스타일 조정
